# 要求落实说明

## 来源要求

依据《全向轮机器人建模问题总结.md / .pdf》中的建议，将模型从“仅 RViz 可视化”整理为“可提交、可编译、可接 Gazebo 与导航接口”的标准 ROS 包。

## 已落实

1. 标准 ROS package：`package.xml`、`CMakeLists.txt`、`launch/`、`urdf/`、`rviz/`、`scripts/`、`config/`、`worlds/`、`meshes/`、`docs/`。
2. RViz：`launch/display.launch` 可一键显示模型。
3. Gazebo：`launch/gazebo.launch` 可加载 world 并 spawn 机器人。
4. 运动插件：URDF 内加入 `libgazebo_ros_diff_drive.so`。
5. 雷达插件：`lidar_link` 上加入 `libgazebo_ros_laser.so`，发布 `/scan`。
6. 导航接口：统一使用 `/cmd_vel`，发布 `/odom` 和 `/tf`。
7. TF：增加 `base_footprint`，形成 `odom -> base_footprint -> base_link -> lidar_link`。
8. 物理属性：主要 link 包含 `inertial` 与简化 `collision`。
9. 速度测试：`scripts/test_cmd_vel.sh` 支持 forward/back/left/right/stop。
10. 差速轮说明：明确 `linear.y` 不适用于两差速轮。

## 室内导航对比

`service_robot_diff_navigation` 使用本包的差速 URDF，将其接入与全向轮相同的 indoor world、地图和五项任务。差速 URDF 中的速度控制板和里程计模块仅从可见模型中隐藏，`/cmd_vel`、`/odom`、`/tf`、`/scan` 等导航接口继续保留。

## 验收命令

```bash
roslaunch sweeper_robot_diff2_submit_ros display.launch
roslaunch sweeper_robot_diff2_submit_ros gazebo.launch
rostopic list
```

应关注话题：

```text
/cmd_vel
/odom
/tf
/scan
/joint_states
/robot_description
```
