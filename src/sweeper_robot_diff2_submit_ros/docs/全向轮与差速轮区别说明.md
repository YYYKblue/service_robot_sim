# 全向轮与两差速轮区别说明

本提交包参考的是《全向轮机器人建模问题总结》中的工程化要求，但机器人驱动形式已经改为“两差速轮”，因此不能把全向轮运动学直接照搬。

## 可以沿用的要求

- 标准 ROS Noetic package 结构；
- RViz 一键显示；
- Gazebo 一键启动；
- `base_footprint` 坐标系；
- `/cmd_vel` 速度接口；
- `/odom`、`/tf`、`/scan` 等导航相关话题；
- 合理的 `inertial` 和 `collision`；
- 便于后续接入 `move_base` 或 SLAM / AMCL。

## 不能照搬的部分

全向轮可以利用 `linear.y` 横向平移，两差速轮不可以。两差速轮只能通过左右轮差速实现前进、后退和转向。

| 项目 | 全向轮机器人 | 两差速轮机器人 |
| --- | --- | --- |
| 前进 / 后退 | 支持 `linear.x` | 支持 `linear.x` |
| 左右横移 | 支持 `linear.y` | 不支持 |
| 原地旋转 | 支持 `angular.z` | 支持 `angular.z` |
| 轮速计算 | 四轮全向运动学 | 左右轮差速运动学 |
| Gazebo 插件建议 | `planar_move` 或自定义全向插件 | `gazebo_ros_diff_drive` |

## 本模型采用的差速运动学

```text
left_wheel  = (linear.x - angular.z * wheel_separation / 2) / wheel_radius
right_wheel = (linear.x + angular.z * wheel_separation / 2) / wheel_radius
```

本模型参数：

```text
wheel_radius = 0.068 m
wheel_separation = 0.396 m
cmd_vel topic = /cmd_vel
```

## 坐标与外观对齐约定

差速导航对比包与全向轮模型使用同一坐标约定：车头为 `+X`，左右轮中心分别为 `y=+0.198 m` 和 `y=-0.198 m`，轮轴沿 `Y` 方向。这样相同的出生姿态、waypoint 和 yaw 才具有相同含义。

差速模型的外壳、上盖、颜色、中心雷达和橙色车头标识与全向轮模型保持一致；差速轮仍保留真实的两轮结构。速度控制板和里程计模块不再作为可见 URDF 几何体出现，但 `/cmd_vel`、`/odom` 和 `/tf` 通信接口继续由 Gazebo 插件提供。

因此验收时不要发送 `linear.y` 作为横移指令；如果发送，差速轮底盘不应该横移。
